Part 1 — Java Source Files

All Activity classes and service/data helpers in com.example.llmlearningassistant

LoginActivity.java

The entry point of the app that presents the login form to the user. It validates credentials and navigates to the Home screen on success, or to Register if the user is new.
Methods
MethodPurpose
onCreate()Inflates the login layout, binds views, attaches the fade-in animation to the card, and sets up click listeners for the Login button and "Need an Account?" link.
attemptLogin()Reads and trims the username and password fields, shows inline errors if empty, checks against dummy credentials (student / password), and navigates to HomeActivity on success or shows a Toast on failure.

RegisterActivity.java

Handles new user account creation by collecting six input fields and validating them before proceeding. On success it navigates forward to the Interest selection screen.
Methods
MethodPurpose
onCreate()Inflates the register layout, binds all six EditText fields and the Create Account button, applies the fade-in animation, and wires the button listener.
onSupportNavigateUp()Handles the system back arrow so the user can return to LoginActivity from the Register screen.
attemptRegister()Validates all six fields (empty checks, email match, password match) and navigates to InterestActivity on success, passing the username as an intent extra.

InterestActivity.java

Displays a scrollable grid of topic chips the user can tap to select their learning interests (up to 10). The selected topics are passed to HomeActivity to personalise the quiz content.
Methods
MethodPurpose
onCreate()Inflates the layout, retrieves the username from the intent, calls buildChips(), and sets up the Next button to pass the selected topics to HomeActivity.
buildChips()Iterates over the 12 topic strings, creates a styled Material Chip for each one, attaches a checked-change listener that enforces the 10-topic maximum, and adds each chip to the ChipGroup.

HomeActivity.java

The main dashboard screen that greets the user, shows the current learning task based on their top selected interest, and provides two LLM-powered learning tools — Generate Hint and Explain Answer — with a custom prompt input field.
Methods
MethodPurpose
onCreate()Reads username and interests from the intent, picks the first interest as currentTopic, populates the task card title and description from QuizData, and wires all three buttons and both LLM utilities.
showLoading()Displays the prompt text and the "Loading…" label, hides the response and error views, and disables both LLM buttons to prevent double-tapping.
showResponse()Hides the loading label, makes the response TextView visible, sets its text, re-enables the buttons, and starts a fade-in animation on the response view.
showError()Hides the loading label, shows the error message in the error TextView, and re-enables both LLM buttons.
onBackPressed()Calls finishAffinity() to close the entire back stack rather than going back to the login screen after a successful login.

QuizActivity.java

Presents three multiple-choice questions for the topic received from HomeActivity. Provides an LLM-powered hint button with a custom prompt input field, validates answers, and passes results to ResultActivity.
Methods
MethodPurpose
onCreate()Reads the topic from the intent, loads its questions from QuizData, calls populateQuestion() for all three cards, and wires the Generate Hint and Submit buttons.
populateQuestion()Sets the question text on the label TextView, clears the RadioGroup, then creates and styles a RadioButton for each answer option and adds it to the group.
submitQuiz()Reads the selected index from each RadioGroup, shows a Toast if any question is unanswered, then bundles question texts, student answers, and correct flags into an intent and starts ResultActivity.
getCheckedIndex()Returns the zero-based index of the selected RadioButton within a given RadioGroup, or -1 if nothing is selected.
showLoading()Displays the prompt text and loading label, hides response and error views, and disables the Generate Hint button.
showResponse()Hides the loading label, displays the LLM hint response with a fade-in animation, and re-enables the hint button.
showError()Hides the loading label, shows the error message, and re-enables the hint button.
onBackPressed()Navigates back to HomeActivity using the default back behaviour with no slide transition.

ResultActivity.java

Displays the student's answers for all three questions with correct/incorrect labels. The Explain Answer button builds a targeted LLM prompt that covers only the questions the student got wrong.
Methods
MethodPurpose
onCreate()Reads questions, answers, and correct flags from the intent, binds all views, calls populateResults(), and wires the Explain Answer and Continue buttons.
populateResults()Dynamically creates a question TextView and an answer TextView for each of the three questions, colouring correct answers in mint-dark green and incorrect ones in error red, and adds them to the result LinearLayout.
explainWrongAnswers()Loops through the correct[] array to collect all wrong questions, builds a specific LLM prompt listing each wrong Q&A pair, and calls LLMService.query(); if all answers are correct it instead asks the LLM to confirm why each is right.
showLoading()Displays the prompt text and loading label, hides response and error views, and disables the Explain Answer button.
showResponse()Hides the loading label, shows the LLM explanation with a fade-in animation, and re-enables the button.
showError()Hides the loading label and response view, shows the error message, and re-enables the button.
onBackPressed()Returns to QuizActivity using the default back behaviour with no slide transition.

LLMService.java

A static service class that handles all HTTP communication with the local Ollama server running on port 11434. Every network call is executed on a new background thread; results are delivered back via a callback interface.
Interface
MemberPurpose
LLMCallbackDefines onSuccess(String) and onError(String) so callers can receive the Ollama result on whichever thread they choose to dispatch to.

Methods
MethodPurpose
generateHint()Prepends a "give a short hint but not the full answer" instruction to the question text and forwards the assembled prompt to callOllama().
explainAnswer()Builds a prompt that includes both the question and the student's answer and asks the model to explain clearly whether it is correct or incorrect.
query()Passes a fully pre-built prompt straight to callOllama() without modification — used by ResultActivity which constructs its own multi-question prompt.
callOllama()Spawns a new Thread, opens an HttpURLConnection to the Ollama API endpoint, POSTs a JSON body with the model name and prompt, reads the response stream, parses the "response" JSON field, and fires onSuccess or onError on the callback.

QuizData.java

A static data store that holds the question bank for all 12 interest topics. Provides helper methods so any Activity can retrieve the correct questions and topic description by name.
Inner Class
MemberPurpose
QuestionImmutable data holder with three public final fields: text (the question string), options (String[] of answer choices), and correctIndex (zero-based index of the right answer).

Methods
MethodPurpose
static initialiserPopulates the BANK HashMap at class load time with three Question objects for each of the 12 topics: Algorithms, Data Structures, Web Development, Testing, Machine Learning, Databases, Networking, Operating Systems, Software Engineering, Mobile Development, Cybersecurity, and Cloud Computing.
getQuestions()Looks up the given topic name in BANK and returns its Question[] array; falls back to the Algorithms set if the topic key is not found.
getDescription()Returns a single descriptive sentence for the given topic used to populate the task card subtitle on the Home screen; falls back to a generic sentence if the topic is not listed.

The app configuration file declaring all activities, permissions, and the launcher entry point. Two key additions beyond the default: the INTERNET permission for Ollama HTTP calls, and usesCleartextTraffic="true" to allow plain HTTP to the local server on port 11434.
DeclarationValuePurpose
